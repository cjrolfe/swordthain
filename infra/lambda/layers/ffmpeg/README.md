This directory is a placeholder — CDK's local asset bundling (see `FfmpegLayer` in `lib/media-app-stack.ts`) generates the actual layer contents at synth/deploy time by installing `@ffmpeg-installer/ffmpeg` via npm's cross-platform install flags (`--os=linux --cpu=x64 --libc=glibc`), same mechanism as `SharpLayer`, then copying the binary into a `bin/ffmpeg` layout — Lambda adds a layer's `bin/` directory to `PATH` automatically. No Docker required. Nothing in this directory is committed except this note.

Previously this downloaded a static build directly from johnvansickle.com at deploy time. That worked, but was a real external-website dependency in the build pipeline — it caused an actual (transient) CI failure when GitHub Actions' runners couldn't reach it. Switched to `@ffmpeg-installer/ffmpeg` since it resolves entirely through npm's own registry via real per-platform `optionalDependencies` (`@ffmpeg-installer/linux-x64`, etc.), the same pattern Sharp uses — which has had zero reliability issues here. Trade-off: this ffmpeg build is from 2021 (4.1.x), not current. More than sufficient for the poster-frame/HEIC-still extraction this project does; revisit if a feature ever needs a newer ffmpeg.
